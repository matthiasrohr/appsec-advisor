## Business Context

This is an application security (AppSec) plugin for Claude Code, designed to help development teams perform automated STRIDE-based threat modeling on their repositories. The plugin operates as a Claude Code plugin (installable via `claude --plugin-dir`) and provides:

1. **Core Product:** Automated threat model generation via a six-agent AI pipeline
2. **Primary Users:** AppSec teams and development teams performing security reviews
3. **Deployment Model:** Local developer tool + CI/CD headless mode
4. **Data Sensitivity:** The plugin processes repository source code locally; source code is transmitted to the Anthropic API for analysis. No persistent user data storage.

## Architecture Notes

**Six-agent pipeline:**
1. `appsec-threat-analyst` (orchestrator, Sonnet, 75 maxTurns)
2. `appsec-context-resolver` (Sonnet, 25 maxTurns) — Phase 1
3. `appsec-recon-scanner` (Sonnet, 25 maxTurns) — Phase 2
4. `appsec-dep-scanner` (Sonnet, 15 maxTurns) — Phase 2 optional
5. `appsec-stride-analyzer` (Sonnet, 31 maxTurns) — Phase 9, per component
6. `appsec-qa-reviewer` (Sonnet, 40 maxTurns) — Stage 2 (skill level)

**Key technologies:**
- Agent definitions: Markdown files with YAML frontmatter
- Hook scripts: Python 3 (security_steering.py, agent_logger.py)
- Validation scripts: Python 3 (validate_intermediate.py, baseline_state.py, qa_checks.py)
- Test framework: pytest
- No web server, no database, no authentication layer
- External API dependency: Anthropic API (Claude models)

**Hook system:**
- UserPromptSubmit: security_steering.py — keyword-based injection
- PreToolUse / PostToolUse / Stop / SubagentStop: agent_logger.py — logging

**Config:**
- plugin/config.json: external_context (disabled), pricing, logging
- skills/check-appsec-requirements/config.json: requirements (disabled)
