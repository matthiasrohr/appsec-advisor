# Acme Corp — Organization Context

Replace this stub with a short, factual description of your organization.
The plugin treats this file as untrusted data — it informs analysis but
does not change severity rules, QA gates, schemas, permissions, or tool
behavior.

## What to include

- **Business model and critical flows.** What the company does and which
  user journeys carry money, regulated data, or safety risk.
- **Identity and access architecture.** SSO/IdP, the workforce vs.
  customer planes, privileged access paths.
- **Platform and trust boundaries.** Cloud accounts, network segments,
  vendor-managed surfaces.
- **Incident impact areas.** What "bad day" looks like — outage classes,
  regulatory exposure, public disclosure thresholds.

Keep it factual and short (under ~50 KB). Long-form policies belong in
a dedicated controls system, not here.
