## Business Context

OWASP Juice Shop is a deliberately insecure web application designed for security training, CTF competitions, and awareness demonstrations. It encompasses vulnerabilities from the entire OWASP Top Ten plus many other real-world security flaws. The application simulates an online juice shop with e-commerce features including user registration, product browsing, ordering, payment, and customer management.

**Important note:** This application is intentionally vulnerable by design. Many of the security findings documented in this threat model are deliberate challenge scenarios. However, the architectural patterns used represent real-world anti-patterns that would be critical findings in a production system.

## Architecture Notes

- **Type:** Full-stack web application (monolith)
- **Backend:** Node.js / Express.js / TypeScript
- **Frontend:** Angular SPA (TypeScript)
- **Database:** SQLite (via Sequelize ORM) + MongoDB (for orders/reviews)
- **Authentication:** Custom JWT implementation using RS256 (RSA keys hardcoded in source)
- **Deployment:** Docker container (distroless base, UID 65532)
- **Port:** 3000 (HTTP, no TLS termination in the app itself)

## Key Security Findings from Code Inspection

### Critical
- RSA private key hardcoded in lib/insecurity.ts:23
- HMAC secret hardcoded in lib/insecurity.ts:44 ('pa4qacea4VK9t9nGv7yZtwmj')
- SQL injection in routes/login.ts:35 (raw string interpolation into SELECT query)
- SQL injection in routes/search.ts:22 (raw string interpolation into SELECT query)
- Server-Side Template Injection / eval() in routes/userProfile.ts:55 (username passed to eval)
- RCE via vm.runInContext + notevil in routes/b2bOrder.ts (eval sandbox)
- XXE in routes/fileUpload.ts (libxml with noent: true)

### High
- JWT stored in localStorage (request.interceptor.ts:13)
- bypassSecurityTrustHtml in 8+ Angular components (systemic XSS)
- CORS wildcard: app.use(cors()) without origin restriction (server.ts:182)
- /ftp directory browsing exposed unauthenticated (server.ts:269)
- /support/logs log file exposure unauthenticated (server.ts:281)
- /encryptionkeys directory browsing (server.ts:277)
- /api-docs Swagger UI exposed publicly (server.ts:286)
- Password change without requiring current password (changePassword.ts:27 — CSRF)
- /metrics Prometheus endpoint (unauthenticated)

### Known Architectural Anti-Patterns
- No BFF pattern: Angular SPA holds JWTs in localStorage
- Custom JWT/auth instead of standard OIDC/SSO
- No secret management: all secrets hardcoded or env-var
- No WAF, no API gateway, direct port 3000 exposure
- Single process hosts both API and static file server
- SQLite in-process database (no network auth)
